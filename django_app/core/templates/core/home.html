MFA System

Multi-Factor Authentication Demo

A Raspberry Pi prototype for tiered physical access control using RFID, fingerprint, and PIN authentication.

Overview

How it works

Users must satisfy a set of authentication factors before gaining access to a protected resource. The required factors depend on the access tier assigned to that resource. Each attempt is logged as an audit event.

Access Tiers

Tier Requirements

Tier Label Required Factors Notes
Tier 1 Basic RFID badge + Fingerprint Highest-complexity normal mode
Tier 2 Elevated RFID badge + PIN Simplified normal mode
Tier 3 High RFID badge + PIN Degraded mission-continuity; restricted to pre-approved resources only
Workflow

Steps

  • Enroll — Register a user's RFID badge, fingerprint, or PIN against their account.
  • Access — Submit an access request for a subject, resource, and tier. The system evaluates enrolled credentials and returns an approved or denied decision.
